# Community-Weather-Oracle

Crowdsourced weather monitoring network with IoT sensors and hyper-local weather prediction services.

Community-Weather-Oracle is a suite of on-chain smart contracts (Clarity) designed to:

- Incentivize submission of accurate weather observations from community-run sensors
- Score contributor accuracy over time with reputation mechanics
- Enable a simple, hyper-local weather data marketplace for consumers (e.g., agriculture, insurance, logistics)

This repository uses Clarinet for development, testing, and deployment workflows.

## Contract Overview

This project contains two core contracts initially developed on a separate development branch:

1. weather-sensor-incentives
   - Purpose: Reward participants for submitting weather observations. Tracks submissions and issues rewards when data meets validity checks.
   - Features: sensor registration, submission tracking, basic anti-spam guardrails, and a simple reward pool.

2. prediction-accuracy-scoring
   - Purpose: Maintain a reputation score for contributors over time based on the accuracy of their reported data versus reference values.
   - Features: rolling accuracy scoring, stake-based credibility weighting, cooldowns, and helper views.

A third contract spec is defined for future expansion:

3. hyper-local-data-marketplace
   - Purpose: Facilitate lightweight purchase access to aggregated, hyper-local weather observations.
   - Features: listing creation by data providers, purchase records, and basic accounting.

Note: Per the requirements, there are no cross-contract calls or trait usage. Each contract is self-contained and designed to be deployed independently.

- .vscode/            Editor tasks and settings for Clarinet

## Development Workflow

The main branch only contains initialization files and documentation. All contract development occurs on the development branch.

Typical flow:

- Create project with Clarinet
- Initialize git and push main with README and scaffolding only
- Branch to development and implement contracts
- Run `clarinet check` regularly
- Open a pull request from development into main

## Getting Started

Prerequisites:
- Node.js LTS
- Clarinet (Hiro): https://docs.hiro.so/clarinet
- Git + GitHub CLI (gh)

Install Clarinet (if not installed):
- macOS/Linux: `npm install -g @hirosystems/clarinet`
- Windows (PowerShell): `npm install -g @hirosystems/clarinet`

Useful commands:
- `clarinet contract new <name>`: Create a new contract scaffold
- `clarinet check`: Validate all contracts under `contracts/`

## Data Model Concepts

- Contributor: Principal that registers a sensor or reports data.
- Observation: Weather data point (e.g., temperature, humidity, precipitation) with location and timestamp.
- Reward Pool: Simple on-chain accounting bucket from which rewards are minted/granted.
- Reputation Score: Rolling measure of contributor accuracy derived from their historical variance vs. reference values.

## Security and Simplicity

- No cross-contract calls
- No trait implementations
- Minimal surface area to satisfy functional requirements
- Simple guards against spam and replay
